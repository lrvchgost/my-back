# API

#### 1. Сущности
** Хранилище (Storage)**

| Поле         | Бизнес-описание                                             | Пример значения             |
|--------------|-------------------------------------------------------------|-----------------------------|
| title        | Заголовок хранилища, String max 256 символов                | "Selecter customers"        |
| description  | Детальное описание хранилища, String max 500 символов       | "Для бесплантых пользоватей"|
| availability | Требуемая доступность хранилища, Int  (0-99.999)           | "99.5"                      |
| capacity     | Объем хранилища, TB                                         | "250"                       |
| paymentType  | Вариант оплаты за улуги, enum("free", "prepaid", "licence") | "free"                      |
| id           | Идентификатор хранилища, Int                                | "10203"                     |
| readSpeed    | Скорость чтения, MБ/с enum(100/150/200)                     | "150"                       |
| writeSpeed   | Скорость записи, MБ/с enum(100/150/200)                     | "150"                       |

#### 2. Бизнес-функции (методы)
**CRUDS для объявлений**
1. **`storage.create`** - Создание нового хранилища
    - *Бизнес-правило*: Только верифицированные пользователи могут создавать хранилища
    - *Ограничения*: Минимум 3 символа в заголовке, все поля обязательны

2. **`storage.update`** - Обновление существующего хранилища
    - *Бизнес-правило*: Только верифицированный пользователь может изменять хранилище

3. **`storage.read`** - Получение информации о хранилище
    - *Бизнес-правило*: Только верифицированный пользователь может читать хранилище

4. **`storage.delete`** - Удаление хранилища
    - *Бизнес-правило*: Возможность удаления только верефицированным пользователем

5. **`storage.search`** - Поиск хранилищ
    - *Бизнес-логика*: Возвращает список хранилищ принадлежащих пользователю с пагинацией
    - *Фильтры*: По емкости, варианту оплаты, скорости чтения и записи, доступности

**Специальные методы**
6. **`storage.optimize`** - Предложение по объединению нескольких хранилищ в одно
    - *Бизнес-логика*: Хранилища могут быть объединены
    - *Критерии совместимости*:
        - Совпадение варианта оплаты
        - Совпадение скоростей записи и чтения
        - Совпадение доступности
        - Срок владения объединенго хранилища равен минимальному сроку среди всех хранилищ
    - *Возвращает*: Список доступных к объединению по указанным параметрам 

#### 3. Бизнес-правила
1. **Ограницение на оптимизацию по варианту оплаты **
    - Не доступно для free
    - Без ограничений для prepaid, licence

2. **Ограничения на максимальное кол-во хранилищ для одного пользователя для разных вариантов оплаты**:
    - Максимум 2 для free
    - Максимум 5 для prepaid
    - Без ограничения для licence

#### 4. Бизнес-метрики успеха
1. **Cкорость оптимизации**: 95% запросов на оптимизацию должны возвращать результаты < 10 сек
2. **Cкорость поиска**: 95% запросов поиска должны возвращать результаты < 2 сек
3. **Конверсия**: 50% новых пользователей имеющих free в течении 3 месяцев переходят на prepaid или licence
4. **Ограничения**:
    - Верифицированные компании: 100 запросов/минуту
